# Руководство по использованию и проверке — Калькулятор национальной кухни

**Настольное приложение** (Python + Tkinter) с базой **SQLite** (`database.db`).  
Используйте этот файл **каждый раз**, когда нужно убедиться, что система работает: следуйте пунктам **по порядку**.

---

## 1) Запуск

1. Откройте терминал (PowerShell или CMD).
2. Перейдите в папку проекта:
   ```powershell
   cd c:\Users\Hossam\Desktop\pythonProject100
   ```
3. Запустите программу:
   ```powershell
   python main.py
   ```
4. Откроется окно входа (может быть развёрнуто на весь экран).

## 2) Учётные записи по умолчанию (для тестов)

| Логин | Пароль | Роль | Что может добавлять |
|-------|--------|------|---------------------|
| `admin` | `123` | Администратор | Рецепт в общий каталог + **управление пользователями** |

## 4) Что видно в главном меню в зависимости от роли

### Все (`user`, `chef`, `admin`)

1. **Каталог рецептов** — просмотр, поиск, фильтры, детали, калькулятор из каталога.  
2. **Мировые кухни** — справочник кухонь.  
3. **Статистика** — отчёты и цифры.  
4. **Калькулятор** — выбор рецепта и пересчёт ингредиентов на число порций/людей.

### Только `user`

5. **Создать рецепт** — **личный** рецепт (в каталоге помечается, в БД тег «персональный»).

### `chef` и `admin`

6. **Добавить рецепт** — блюдо/рецепт в **общий каталог** (основная база).

### Только `admin`

7. **Управление** — учётные записи пользователей.

## 6) Как добавить «блюдо» (рецепт) — два варианта

### А) Блюдо в общий каталог (видят все пользователи)

**Кто может:** только `admin` или `chef`.

1. Войдите как **`chef`** или **`admin`**.  
2. В главном меню нажмите **«➕ Добавить рецепт»** (или арабский «إضافة وصفة» при языке AR).  
3. Заполните поля (подписи в форме обычно на русском):

   | Поле | Обязательно? | Замечание |
   |------|----------------|-----------|
   | Название рецепта | Да | Лучше уникальное и понятное имя |
   | Описание | Желательно | Технически может быть пустым, но лучше заполнить |
   | Кухня | Да | Выберите из списка **точное имя кухни** из справочника |
   | Категория | Необязательно | салат, суп, закуска, основное блюдо, десерт, напиток или «Без категории» |
   | Ингредиенты | Да | Каждая строка: `название|количество|единица` (см. раздел 7) |
   | Инструкции | Да | Текст шагов приготовления |
   | Порции | Да | Целое число > 0 |
   | Сложность | Да | Одно из: `легко`, `средне`, `сложно` |

4. Нажмите **💾 Сохранить**.  
5. При успехе появится сообщение, затем возврат в главное меню.  
6. Проверка: **Каталог рецептов** → **Поиск** — новый рецепт должен быть в списке (при необходимости отфильтруйте по кухне).

### Б) Личный рецепт (обычный пользователь)

**Кто может:** только `user` (кнопка **«Создать рецепт»** в главном меню).

1. Войдите как **`user`**.  
2. **«✍️ Создать рецепт»** (или арабский вариант).  
3. Те же идеи полей: название, описание, кухня из списка, ингредиенты (`название|количество|единица`), инструкции, порции, сложность.  
4. Сохранение — рецепт помечается как **персональный**, в каталоге часто отображается со **★** в названии.  
5. **Важно про кухню в личном рецепте:** в списке могут быть строки вида «Греческая кухня»; связь с записью в БД идёт через поиск по шаблону — выбирайте значение из списка **как показано**.

---

## 7) Формат ингредиентов (очень важно)

В **«Добавить рецепт»** и в **личном рецепте** ингредиенты задаются **построчно**:

```text
название|количество|единица
```

Пример:

```text
Курица|500|г
Рис|200|г
Лук|1|шт
```

- Разделитель полей: **`|`** (вертикальная черта).  
- Не оставляйте пустыми обязательные поля — иначе появится сообщение об ошибке.

## 8) Каталог рецептов — фильтры и быстрая проверка

1. Откройте **Каталог рецептов**.  
2. **Поиск по имени:** введите часть названия → **Поиск** 🔍.  
3. **Кухня:** выберите кухню в списке → **Поиск**.  
4. **Сложность:** `легко` / `средне` / `сложно` → **Поиск**.  
5. **Категория:** например `суп` ищет **вхождение** в поле тегов (например совпадёт с `суп,говядина`).  
6. **Один щелчок** по строке таблицы — выделение рецепта.  
7. **Просмотр** — карточка деталей должна открываться без ошибки при актуальной БД (колонка `image_path` есть).  
8. **Рассчитать** — открывается калькулятор для этого рецепта.  
9. Выход: **Назад**, затем **Назад в главное меню** по завершении.

---

## 9) Полный сценарий проверки «каждый раз» — строго по шагам

Выполните **от начала до конца** один раз; если нет неожиданных ошибок — система в порядке.

### Этап А — пользователь (`user` / `000`)

1. Запустите `python main.py`.  
2. (По желанию) нажмите **🇸🇦** для арабского интерфейса.  
3. Логин: `user`, пароль: `000` → **Войти**.  
4. Подтвердите приветственное окно **OK**, если оно появилось.  
5. **Каталог рецептов** → **Поиск** → выберите разные кухни и **Поиск** минимум дважды → выделите строку → **Просмотр** → **Назад** в каталог → **Рассчитать**, затем в главное меню.  
6. **Мировые кухни** → просмотр → **Назад в главное меню**.  
7. **Статистика** → просмотр → **Назад в главное меню**.  
8. **Калькулятор** → выберите рецепт, при наличии полей измените число порций → **Назад в главное меню**.  
9. **Создать рецепт** — введите простой тестовый рецепт (ингредиенты по формату выше) → **Сохранить**.  
10. Снова каталог — личный рецепт должен отображаться (часто со **★**).  
11. **Выход из аккаунта**.

### Этап Б — шеф-повар (`chef` / `111`)

1. **Войти** как `chef` / `111`.  
2. Повторите шаги 5–8 этапа А (каталог, кухни, статистика, калькулятор).  
3. **Добавить рецепт** — добавьте тестовый рецепт в общий каталог → **Сохранить**.  
4. **Каталог рецептов** → **Поиск** — убедитесь, что новый рецепт есть.  
5. **Выход из аккаунта**.

### Этап В — администратор (`admin` / `123`)

1. **Войти** как `admin` / `123`.  
2. Кратко проверьте **каталог** и **калькулятор** (желательно).  
3. **Управление** — откройте список пользователей, только просмотр (не удаляйте учётки по умолчанию без необходимости) → **Назад** в главное меню.  
4. **Добавить рецепт** (по желанию) или пропустите.  
5. **Выход из аккаунта**.

**После этапов А + Б + В:** отсутствие неожиданных ошибок означает согласованную работу входа, прав, каталога и калькулятора.

## 10) Частые проблемы

| Проблема | Вероятная причина | Что делать |
|----------|-------------------|------------|
| Ошибка `no such column: image_path` | Старая база до обновления | Запустите приложение из обновлённого проекта — при старте выполняется миграция и добавляется колонка |
| По выбранной кухне нет рецептов | Нет записей с этой `cuisine_id` | Добавьте рецепт с этой кухней или используйте встроенные данные `seed` проекта |
| Фильтр категории не даёт строк | В тегах нет выбранного слова | Другая категория или пустая категория + поиск по кухне |
| Нельзя добавить в общий каталог | Вы вошли как `user` | Войдите как `chef` или `admin` |
